# Definition of Done

## Business

- [ ] Business requirements are fully implemented.
- [ ] Business rules are enforced.

## Architecture

- [ ] The implementation follows the project architecture.
- [ ] Responsibilities are properly separated.

## Code Quality

- [ ] The code follows the coding conventions.
- [ ] SOLID principles have been respected where applicable.
- [ ] No unnecessary complexity has been introduced.

## Testing

- [ ] Unit Tests added (when applicable).
- [ ] Feature Tests added (when applicable).
- [ ] All tests pass.

## Documentation

- [ ] OpenAPI documentation updated.
- [ ] Project documentation updated.
- [ ] ADR created (if an architectural decision was introduced).

## Version Control

- [ ] Changes committed.
- [ ] Commit message follows Conventional Commits.
